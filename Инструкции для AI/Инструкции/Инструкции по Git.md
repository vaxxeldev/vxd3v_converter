ПРАВИЛА ПО ЛОКАЛЬНОЙ ИСТОРИИ (git, без пушей):

- Веди частую локальную историю коммитов — коммить каждый завершённый
  и согласованный шаг, даже маленький. Не жди "большого" изменения.
- Локальные коммиты делай свободно, без отдельного "да" на каждый,
  если сам шаг был согласован в плане. Коммит — это фиксация
  согласованного результата, а не новое решение.
- Сообщение коммита: короткое, на английском, в стиле conventional commits
  (fix:, feat:, refactor:, test:, chore:), по сути изменения.
- НИКОГДА не взаимодействуй с удалённым репозиторием (origin) без
  явного разрешения: ни git push, ни git pull, ни git fetch с изменением
  локальных веток, ни force push. Локальная история — твоя черновая
  рабочая зона, github трогаем отдельно и осознанно.
- Если нужно "откатиться" — используй git log / git diff, чтобы показать
  мне варианты, и спроси перед любым reset/checkout, который удалит
  несохранённые изменения.
- Веди также короткий devlog в файле (например DEVLOG.md или
  .agent/devlog.md) — после каждого шага добавляй 2-4 строки:
  дата, что сделано, какие файлы, почему именно так. Это не коммит-история,
  а человекочитаемый журнал решений, который я могу быстро просмотреть.
- Не переписывай и не сжимай историю коммитов (никаких rebase -i,
  squash, amend на старых коммитах) без явного разрешения.

Цель: я могу в любой момент посмотреть git log и DEVLOG.md и понять,
что происходило, без необходимости спрашивать тебя каждый раз.
